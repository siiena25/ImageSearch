Visual Product Search
On-Device Image Search for Marketplaces

Foundation models fail to distinguish a floral print from a polka-dot one. This project rebuilds the retrieval pipeline around that gap — and compresses the whole stack into an on-device Android demo.

⬡ GitHub Repository 📄 Poster (PDF) ▶ Watch Demo

Abstract

Marketplace shoppers often know what they want visually but struggle to describe it in text. This project builds an end-to-end image search pipeline that accepts a product photo, removes the background and any human model using U2Net-P, routes the query to the right domain expert (FashionSigLIP for clothing, SigLIP 2 otherwise) via a zero-shot category router, retrieves top-50 candidates with cosine similarity, and reranks them using DINOv2 texture features and an HSV colour signature. The complete pipeline runs fully on-device on Android — a blueprint for scaling to a real marketplace backend.

Pipeline

0
User takes a photo In-app camera or gallery — any product on any background.
↓
1
U2Net-P Segmentation Encoder–decoder saliency model produces a foreground mask. Threshold → RGBA crop with transparent background. Chosen over SAM 2 for its 4 MB footprint and parameter-free inference.
↓
2
Zero-Shot Category Router SigLIP 2 text-tower embeds 4 prompts per category. Cosine similarity between the crop embedding and each text prototype selects top-1 / top-2 categories — no fine-tuning needed.
↓
3
Semantic Embedding FashionSigLIP (fashion categories) or SigLIP 2 (other). Input: 224×224 RGB tensor. Output: 768-dim L2-normalised vector.
↓
4
Cosine Retrieval — top 50 Dot-product against pre-computed catalog embeddings filtered by category. Catalog loaded once into memory at app start.
↓
5
Texture + Colour Rerank — top 10 DINOv2-S patch features for texture similarity. 26-bin HSV colour signature per item. Triple rerank (semantic + texture + colour) + hard colour gate → final top 10.

Scaling to Production

This demo runs fully on-device with ~700 items. For a real marketplace (millions of products, dozens of categories):

  • 🖥️  Replace on-device runtime with a backend service (FastAPI / Triton Inference Server)
  • 🗄️  Use a vector database (Faiss, Qdrant, Weaviate) for sub-millisecond ANN search
  • 📐  Add HNSW / IVF indexing for large-scale recall
  • 🏷️  Expand category taxonomy; fine-tune embeddings per vertical as needed
  • 🔄  Incremental index updates as new products are listed
